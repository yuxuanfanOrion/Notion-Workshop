### gpt-oss harmony format





- **计算成本高**：多模态处理（尤其是图像、视频解析）需大量算力，导致多数学术原型（如 CoSearchAgent、Storm）暂未支持，仅商业产品与头部开源项目（如 Qwen、DeepSeek 系列）具备该能力。

- **模态对齐难**：不同模态数据的语义映射存在歧义（如图像中的 “趋势图” 与文本中的 “增长数据” 需精准匹配），现有系统易出现 “模态信息脱节”（如图表数据与文本分析矛盾）。

- **工具集成复杂**：多模态工具的 API 接口差异大，需 MCP 等协议进一步标准化，避免 “多模态工具调用混乱” 问题。





- 直接critic model 训练

- **多模态基准建设**：当前基准（如 HotpotQA、GAIA）聚焦文本任务，需开发 “多模态研究任务基准”（如 “基于论文 PDF 的图像 + 文本，复现实验结论”），评估 DR Agents 的跨模态推理能力。

- **轻量化多模态模型**：通过模型压缩（如稀疏注意力）、模态蒸馏（将多模态知识迁移到轻量模型），降低多模态处理的算力门槛，推动学术与中小企业应用。

- **自进化多模态工具链**：参考 Alita（2025.5 发布）的 “动态 MCP 服务器” 思路，让 DR Agents 自主适配新的多模态工具（如实时接入新的图像分析 API），无需人工更新架构。\

- **self-evolution / self-play **
  - ALITA

- “可执行监督”到“因果奖励”：将“证据可回溯性”纳入奖励（如要求回答必须引用可验证的网页段落/截图），减少投机路径。

- 分层策略学习：高层做计划与子任务分解，低层做页面操作与视觉读写；配合选工具/选页面/选证据的子策略蒸馏。

- 反事实/自博弈数据：让代理在相同问题下尝试不同工具序列，比较性训练学到更鲁棒的决策。

- 更强的多模态数据构造
  - 多图多页组合任务：需要跨多张图像、多个网页与时间轴汇总的时序追踪（新闻、项目进度）。

  - 过程型问题：不仅给最终答案，还要求提交证据链、引用定位与可复现脚本。

  - 合成与真实混合：用真实网页+合成干扰（错误图、伪链接、过时数据）评测鲁棒性与事实核验能力。

- 分层奖励建模，轨迹level验证器

- 回复增强
